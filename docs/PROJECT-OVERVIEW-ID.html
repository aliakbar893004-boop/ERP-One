1. Apa itu ErpOne?

ErpOne adalah aplikasi ERP berbasis web untuk bisnis ritel & distribusi (konteks UMKM/distributor Indonesia). Mencakup data master, pembelian, penjualan, gudang/persediaan, kasir (POS), keuangan (Piutang/Hutang/Biaya), laporan, dan dashboard operasional. Antarmuka memadukan label Inggris dengan istilah bisnis Indonesia.

Teknologi

.NET 10 Blazor Server (InteractiveServer) EF Core SQL Server FluentValidation ClosedXML (Excel) QuestPDF (PDF) Bootstrap + Bootstrap Icons xUnit (+ SQLite in-memory) ASP.NET Identity

2. Arsitektur (Clean Architecture)

Empat project dengan ketergantungan mengarah ke dalam. Layer luar bergantung pada layer dalam; Domain tidak bergantung pada apa pun.

LayerProjectTanggung jawabBergantung pada
DomainErpOne.DomainEntity, enum, aturan domain. Tanpa dependensi framework.—
ApplicationErpOne.ApplicationInterface service, DTO, validator FluentValidation, kontrak/port.Domain
InfrastructureErpOne.InfrastructureEF Core AppDbContext, implementasi service, migrasi, wiring DI.Application, Domain
WebErpOne.WebHalaman/komponen Blazor, otorisasi, menu, seeding, tampilan (appearance).Application, Infrastructure

Pengujian: ErpOne.UnitTests (logika domain) dan ErpOne.IntegrationTests (service di atas SQLite; mereferensi ErpOne.Web lewat WebApplicationFactory<Program>).

3. Organisasi folder (per header menu)

Di setiap project, file dikelompokkan ke subfolder yang mengikuti header menu sidebar. Folder hanya untuk penataan — namespace TIDAK di-nest (mis. service di Services/Master/ tetap di namespace ErpOne.Infrastructure.Services). Ini menjaga build tanpa friksi sekaligus memberi peta yang jelas.

Master/        Product, ProductCategory, Brand, Unit, Attribute, Warehouse,
              Tax, PaymentMethod, Supplier, Customer, Currency
Inventory/     Stock (StockMovement, ProductStock)
Transactions/  PurchaseOrder, GoodsReceipt, SalesOrder, DeliveryOrder
Cashier/       CashierShift, PosSale
Finance/       CashBank, Faktur Supplier/Customer, Pembayaran/Penerimaan, Expense
Reports/       StockLedger, InventoryValuation, Sales, Purchase, GrossProfit
Dashboard/     Dashboard
Settings/      CompanySetting, ApprovalChain, NumberSequence, DocumentNumber, Log

Berlaku untuk Domain/Entities/*, Application/* (folder fitur di-nest di bawah header), dan Infrastructure/Services/*. Application/Common, Reports, dan Dashboard tetap di root project-nya.

4. Data & persistence

  • DbContext: ErpOne.Infrastructure.Persistence.AppDbContext. Entity dikonfigurasi inline di OnModelCreating (blok per-entity).
  • Penamaan tabel: tabel fisik memakai prefiks domain — M_* (master, mis. M_ProductVariants), T_* (transaksi, mis. T_DeliveryOrders).
  • Migrasi: dotnet ef migrations add <Nama> --project src/ErpOne.Infrastructure --startup-project src/ErpOne.Web, lalu dotnet ef database update ....
  • Koneksi: ConnectionStrings:Default di appsettings.json → database SQL Server MyApp.
  • Perhitungan HPP: stok memakai moving-average; COGS pada pengiriman di-snapshot dari ProductVariant.CostPrice saat Post (baris draft menampilkan 0 sampai diposting).

5. Sistem menu & navigasi

Sidebar kiri (icon rail terang, membentang saat hover) bersifat data-driven dari satu registry.

  • AppMenus.cs (Web/Authorization) — mendeklarasikan ResourceGroup (header menu) yang masing-masing berisi AppResource. Tiap resource = (key, label, icon, actions[]). Action berupa AppAction seperti index/create/edit/delete/approve/post/void/export.
  • NavMenuBuilder.cs — membangun sidebar secara konvensi: Href = key.Replace('.', '/'); gate visibilitas = {key}.index; ikon = ikon deklarasi tanpa -fill. Kasus tak beraturan ditangani tabel Overrides (Href/action khusus / selalu tampil / item tambahan).
  • Permission otomatis diturunkan: AppMenus.AllPermissions = seluruh {resource}.{action}. Helper: AppMenus.Perm("master.brands","create").

6. Otorisasi & permission

  • Nama policy = string permission = {resource}.{action} (mis. finance.ar-invoices.index).
  • FallbackPolicy global mewajibkan user terautentikasi (setiap halaman butuh login kecuali [AllowAnonymous]).
  • Halaman dikunci dengan @attribute [Authorize(Policy = "…")]; tombol/kartu dengan <AuthorizeView Policy="…">; form mengecek ulang permission create/edit di OnInitializedAsync.
  • BootstrapSeeder.cs (Web/Infrastructure) memberikan semua AppMenus.AllPermissions ke role admin saat startup (idempoten) dan memastikan user admin bootstrap. Setelah menambah permission, restart + sign out/in agar klaim admin ter-refresh.

10. Build, jalankan & test

# Build / test seluruh solusi
dotnet build ErpOne.slnx
dotnet test  ErpOne.slnx

# Menjalankan: dari Visual Studio (IIS Express) atau
dotnet run --project src/ErpOne.Web
Gotcha: saat app berjalan di Visual Studio/IIS Express, DLL ErpOne.Web terkunci, sehingga dotnet build/test seluruh solusi gagal dengan error copy (MSB3021) — bukan error kompilasi. Test integrasi mereferensi ErpOne.Web, jadi hentikan app yang berjalan sebelum dotnet test. Build Domain/Application/Infrastructure per-project tetap bisa saat app jalan.

11. Konvensi & gotcha

  • Git: commit / merge / push dilakukan manual oleh developer. Otomasi meninggalkan perubahan di working tree (rename memakai git mv).
  • Translasi query: test integrasi berjalan di SQLite — hindari translasi khusus SQL Server (mis. EF.Functions.DateDiffDay). Pakai perbandingan ambang tanggal atau agregasi di memori untuk set kecil.
  • Rupiah: nominal ditampilkan dengan ToString("N0")/N2. (Format locale Indonesia adalah kandidat peningkatan berikutnya.)
  • Test penomoran: saat menambah NumberSequence, naikkan assert jumlah di NumberSequenceServiceTests.
  • File solusi adalah ErpOne.slnx (bukan .sln).
